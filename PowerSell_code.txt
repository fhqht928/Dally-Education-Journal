
ls : 현재 폴더 안에 있는 모든 파일 리스트 출력

start-transcript : 기록 시작

git --version : 깃 허브 버전확인

git clone 깃허브 주소 : 해당 깃허브와 연결

md : make directory (새폴더 만들기)

cd : 해당 폴더로 이동

폴더 이름 일부분 + tab : 해당 폴더 자동 입력

git config --global user.name "LeeSeungHyun" 글로벌 : 사용자를 깃허브에 바로 연결하는 명령어
                        user.email
type 

notepad git_test.py : 해당 파일 노트패드로 열기.(없으면 새로 생성)

cd .. : 이전 폴더로 이동

git status : 스테이터스 확인

git add file : 해당 파일을 버전에 추가해라.  <- 버전관리 시작

git commit -m "Dally Journal" :코밋 작성

git help status : 명령어 도움말창

git push : 깃허브에 올리기.

git fetch : 서버에서 정보 가져오기

git diff HEAD origin/master 현재 컴퓨터 내 파일과 서버 내 파일이 동일한지 점검.

git checkout -b branch_0 브렌치_0이라는 이름의 브렌치 생성

git checkout branch_0 브렌치_0으로 이동

git branch -d branch_0 브렌치_0 삭제
